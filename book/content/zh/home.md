## 这本书写给谁

**基础和前沿都讲。** 一头是想入门的新人，一头是想搞清楚"agent 时代到底变了什么"的资深 SDE、技术负责人、架构师——这本书两头都接。

软件工程的基础（需求、架构、契约、质量、测试、版本控制、CI、项目管理）会**从头讲清楚、不跳过**，但不照本宣科：每个概念先给一个**生活类比**（买菜、点外卖、雇新外包、冰箱上贴家规便条），让你从直觉带入，讲透之后顺势接上"**这个老概念在 AI 时代怎么变、怎么降级、反而怎么更重要**"。资深读者可以快速略过基础层、直取前沿。专业和易读不矛盾——这正是好教材该有的样子。

## 三段式：每个概念怎么讲

> **生活类比 → 工程定义 → 真实案例。**
> 先用一个详实的日常场景把直觉讲透，再给准确定义，再落到带真实数字、真实工件的工程例子。每章按九段骨架走：钩子 / 类比 / 定义 / 为什么重要 / 实践方法 / 真实案例 / 反模式 / 检查清单 / 自测。

## 来源纪律

每条事实都挂一个**来源 ID**，可点开查证。颜色区分级别：<span style="color:#60a5fa">GitHub</span> / <span style="color:#34d399">OpenAI</span> / <span style="color:#c084fc">Anthropic</span> 是**一手**（厂商对自家产品的事实陈述）；<span style="color:#fbbf24">S 开头</span>是**二手分析**（Willison、Karpathy、Yegge、METR 等非厂商）。带"趋势判断"的话是综合分析，不代表任何厂商原文。截止本书写作（访问日期 2026-06-13），产品迭代很快，所有一手事实都经过联网核查——包括发现旧教材引用过一个**根本不存在的 404 链接**。

## 全书地图（10 卷）

1. **格局重置** —— SDE 的工作对象变了：代码不再是中心产物；什么过时、降级、反而更重要；角色之争；三条厂商路线如何汇合；前沿工程原则。
2. **概念重校** —— 经典 SE 在 agent 时代的重定价（需求 / 架构 / 质量 / 测试）。
3. **新工程资产** —— AGENTS.md、context 工程、状态与交接工件、tool/MCP 工程、Agent Skills。
4. **Harness 工程专章** —— 定义、9 层参考架构、sandbox 与安全、eval、trace、成熟度模型。
5. **Agents 工程专章** —— agent 是工作单元；多 agent 模式；managed agents 与 subagents；反模式。
6. **三大 coding agent 实战对比** —— Claude Code / Codex / Copilot；写 agent-ready issue 与指令；review 与 rollback。
7. **Agentic 项目管理** —— 从排人天到排 agent-ready work；新度量与失败分类学。
8. **学习路径与 Capstone** —— 按模块学；给真实 repo 建一套 harness。
9. **模板附录** —— 全部可复制的工程模板。
10. **来源索引** —— 一手与二手分列，带日期与出处。
